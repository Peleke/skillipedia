## Runlet — ND-Adapted Daily Task Compiler

Turn a braindump into tomorrow's runlist. Call at end of session or when the user says 'runlet' / 'braindump' / 'what should I do tomorrow'. Input: unstructured text + optional previous runlist. Output: 4-quadrant Obsidian markdown (~800 tokens) with entry points on high-activation items, carry-forward tags, kill reasoning, and a machine-readable RUNLET_SUMMARY JSON block for agent pings. ND-adapted: classifies by activation energy, not urgency.

## The Matrix

Two axes built for ND cognition, not neurotypical assumptions.

**Axis 1: Energy Cost** — how hard is it to START this task? Not difficulty. Not duration. The activation barrier. Universal across all goals.

**Axis 2: Moves [Focus]** — does this advance the current primary objective within 30 days? Binary. Configurable: Money, Career, Health, Launch, or any user-defined focus.

|              | Low Energy Cost | High Energy Cost |
|-------------|----------------|-----------------|
| **Moves [Focus]** | **Do First** — easy to start, payoff is real | **Block Time** — needs ramp-up, but it's the work that pays |
| **Doesn't Move [Focus]** | **Batch** — low-drag maintenance, do in a clump | **Kill or Delegate** — hard AND doesn't pay? why? |

## Key Features

- **Entry points** on every Block Time item — one sentence describing the smallest physical action to start the task
- **Carry-forward** — unchecked items from yesterday's runlist carry with tags; 3 carries forces a decision
- **Kill reasoning** — strikethrough + reason on every killed item (the brain needs closure, not silence)
- **Focus swap** — change the primary focus mid-day to re-sort without rebuilding
- **Machine-readable summary** — RUNLET_SUMMARY JSON block for agent pings (morning notification, nightly recap)

## Input

Unstructured text: braindumps, session logs, chat transcripts, scattered thoughts. Optionally: yesterday's runlist for carry-forward. Focus axis and date.

## Output

Obsidian-ready markdown with four quadrants, entry points, batch grouping, kill reasoning, and embedded JSON summary. Review time: 2 minutes or less.
